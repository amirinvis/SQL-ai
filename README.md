# دستیار هوشمند SQL Server & Power BI

یک اپلیکیشن وب با UI تیره و حرفه‌ای که با استفاده از Mistral AI فقط به سوالات مربوط به SQL Server و Power BI پاسخ می‌دهد، و امکان آپلود فایل‌های .sql, .dax, .m, .csv, .json و .pbix را دارد.

## ⚠️ نکته امنیتی مهم (حتماً بخوانید)

هرگز کلید API را مستقیم داخل کد فرانت‌اند (HTML/JS که در مرورگر اجرا می‌شود) قرار ندهید؛ چون هرکسی که صفحه را باز کند می‌تواند آن را ببیند و از اعتبار شما مصرف کند. به همین دلیل این پروژه یک **بک‌اند Node.js** دارد که کلید فقط روی سرور (در فایل `.env`) نگه‌داری می‌شود و هرگز به مرورگر ارسال نمی‌شود.

## نصب و اجرا
اول سه فایل app.js و index.html و style.css را در یک پوشه به اسم public بریزید

بعد برید به ترمینال و ادامه مراحل زیر:
```bash
cd sql-bi-assistant
npm install
cp .env.example .env
```

سپس فایل `.env` را باز کرده و کلید جدید Mistral را وارد کنید:

```
MISTRAL_API_KEY=کلید_جدید_شما
```

اجرا:

```bash
npm start
```

سپس مرورگر را باز کنید: `http://localhost:3000`

## قابلیت جدید: درک تصویر (عکس طرح/mockup/ER Diagram)

حالا می‌توانید عکس (`.png` `.jpg` `.jpeg` `.webp` `.gif`) از پنل کناری آپلود کنید — مثلاً:
- طرح دستی یا wireframe یک فرم/گزارش
- دیاگرام ER (روابط بین جداول)
- اسکرین‌شات یک گزارش یا سیستم موجود

عکس در نوار بالای باکس پیام (پیش‌نمایش) نشان داده می‌شود؛ کافیست توضیح یا سوال خود را بنویسید (یا خالی بگذارید تا پیش‌فرض «این تصویر را تحلیل کن...» استفاده شود) و ارسال کنید. دستیار بر اساس تحلیل تصویر:
1. اسکریپت کامل T-SQL برای ساخت جداول در SQL Server پیشنهاد می‌دهد.
2. راهنمای ساخت مدل و ویژوال‌های مناسب در Power BI ارائه می‌کند.

**نکته فنی:** وقتی پیامی شامل عکس باشد، سرور به‌طور خودکار مدل را از `mistral-small-latest` به یک مدل دارای قابلیت **vision** (فعلاً `mistral-medium-latest`) تغییر می‌دهد، چون مدل‌های متنی ساده نمی‌توانند تصویر را ببینند. (مدل قبلی پیشنهادی، `pixtral-large-latest`، توسط Mistral منسوخ (deprecated) و جایگزینش `mistral-medium-latest` شده است.) مدل‌های vision در Mistral به مرور زمان تغییر می‌کنند؛ اگر باز هم خطای «Invalid model» گرفتید، لیست مدل‌های فعلی دارای قابلیت vision را در [مستندات Mistral](https://docs.mistral.ai/capabilities/vision) یا [فهرست مدل‌ها](https://docs.mistral.ai/models/overview) چک کنید و نام مدل را در `server.js` (داخل روت `/api/chat`، متغیر `model`) به‌روزرسانی کنید.

## قابلیت جدید: چت‌های چندگانه و حافظه دائمی

حالا برنامه چند گفتگوی جداگانه رو پشتیبانی می‌کنه:
- دکمه **«چت جدید»** بالای پنل کناری یک گفتگوی تازه می‌سازه.
- زیرش لیست گفتگوهای قبلی نشون داده می‌شه؛ روی هرکدوم کلیک کنید تا بازش کنید، و با × کنار هر آیتم (هنگام هاور) می‌تونید حذفش کنید.
- **«پاک‌سازی این چت»** فقط پیام‌های همون گفتگوی فعلی رو خالی می‌کنه (بدون حذف کل گفتگو از لیست).

**نکته فنی:** این قابلیت با `localStorage` مرورگر پیاده شده — یعنی داده‌ها روی همون مرورگر و دستگاه ذخیره می‌شن، نه روی سرور. پس اگه سرور (`npm start`) رو ببندید و دوباره باز کنید، چون داده در مرورگره نه سرور، گفتگوها باقی می‌مونن. اما اگه مرورگر رو عوض کنید یا از یه دستگاه دیگه وارد بشید، گفتگوهای قبلی رو نمی‌بینید (چون هرکدوم `localStorage` جدا دارن).

**محدودیت مهم:** `localStorage` معمولاً حدود ۵ تا ۱۰ مگابایت ظرفیت داره. چون عکس‌های آپلودی به‌صورت base64 داخل تاریخچه گفتگو ذخیره می‌شن، اگه چند گفتگوی حاوی عکس داشته باشید ممکنه این ظرفیت پر بشه. در این حالت برنامه یه پیام هشدار نشون می‌ده و پیشنهاد می‌کنه گفتگوهای قدیمی‌تر رو حذف کنید. برای استفاده حرفه‌ای‌تر و بدون این محدودیت (مثلاً اگه چند نفر با اکانت جدا از برنامه استفاده کنن)، باید ذخیره‌سازی رو به یک دیتابیس واقعی سمت سرور (مثل SQLite یا حتی همون SQL Server!) منتقل کرد.

## نکات درباره فایل‌های Power BI (.pbix)

فایل‌های `.pbix` در واقع یک فایل ZIP هستند. این ابزار سعی می‌کند فهرست بخش‌های داخلی و در صورت امکان بخشی از Layout گزارش را استخراج کند، اما در نسخه‌های جدید Power BI، مدل داده به‌صورت فشرده باینری (VertiPaq) ذخیره می‌شود که استخراج کامل DAX/جداول از آن با ابزار ساده ممکن نیست.

**پیشنهاد برای بهترین نتیجه:** کوئری‌های Power Query (M) یا فرمول‌های DAX را از Power BI Desktop کپی و در یک فایل `.txt` یا `.dax`/`.m` ذخیره کرده و همان را آپلود کنید — دستیار می‌تواند این‌ها را کامل تحلیل کند.

## قابل توسعه

- افزودن استریم پاسخ (streaming) برای نمایش تدریجی متن
- ذخیره تاریخچه گفتگو در دیتابیس برای چند کاربر
- افزودن احراز هویت اگر قرار است داخل شرکت به‌صورت عمومی در دسترس باشد
